# 과제 1

- 얕은 복사의 장점, 단점을 쓰시오.
  - 장점
 
    **속도가 매우 빠르다**: 객체의 참조(reference)만 복사하기 때문에, 복사 시간이 짧고 메모리 사용량이 적다.
  
    **메모리 효율성이 높다:** 원본 객체와 복사본이 내부 객체를 공유하므로, 불필요한 메모리 중복이 발생하지 않는다.
  
    **간단하게 구현할 수 있다:** 대부분의 언어에서 기본적으로 제공되는 복사 방법(Object.clone() 등)으로 쉽게 사용할 수 있다.  

  - 단점

    **원본과 복사본이 내부 객체를 공유한다:** 중첩된 객체(리스트 안에 리스트, 객체 안에 객체 등)가 있을 때, 한 쪽에서 내부 객체를 수정하면 다른 쪽에도 영향을 미친다.  

    **예측하기 어려운 버그가 발생하기 쉽다:** 특히 큰 객체 구조나 팀 프로젝트에서 디버깅이 힘들어질 수 있다.  


- 깊은 복사의 장점, 단점을 쓰시오.
  - 장점

    **완전한 독립성을 보장한다:** 원본 객체와 복사본이 완전히 별개의 객체를 가지므로, 한 쪽을 수정해도 다른 쪽에 전혀 영향을 주지 않는다.
  
    **안전하다:** 복잡한 객체 구조(중첩된 리스트, 딕셔너리, 클래스 등)에서 데이터 무결성을 유지하기 좋다.
  
    **버그 발생 가능성이 낮다:** 특히 상태를 독립적으로 관리해야 하는 경우(게임 상태 저장, 설정 객체 복사 등)에 매우 유용하다.  

  - 단점

    **속도가 느리다:** 모든 중첩 객체까지 재귀적으로 복사해야 하므로, 얕은 복사에 비해 성능 오버헤드가 크다.
  
    **메모리 사용량이 많다:** 원본의 모든 데이터를 새로 만들어 복사하기 때문에 메모리를 더 많이 소비한다.
  
    **순환 참조(Circular Reference)가 있는 경우 문제가 발생할 수 있다:** 무한 재귀에 빠질 위험이 있어, 특별한 처리가 필요하다.
