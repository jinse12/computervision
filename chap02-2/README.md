## **1. 비주얼 스튜디오 프로젝트 속성에서 Debug 모드, Release 모드의 차이를 설명하라.**
  - Debug 모드는 프로그램을 개발하고 오류를 찾기 위해 사용하는 빌드 모드입니다. 이 모드에서는 디버깅 정보를 포함하고 코드 최적화를 최소화하여 프로그램의 실행 속도는 다소 느릴 수 있지만, 오류를 찾고 수정하기가 쉽습니다.

  - Release 모드는 프로그램을 실제로 배포하기 위해 사용하는 빌드 모드입니다. 이 모드에서는 디버깅 정보를 제거하고 코드 최적화를 수행하여 실행 속도가 빠르고 프로그램의 크기가 작아집니다.

## **2. 비주얼 스튜디오 프로젝트 속성에서 x86 모드, x64 모드의 차이를 설명하라.**
  - x86 모드는 32비트 환경에서 실행되는 프로그램을 만들기 위한 모드입니다. 32비트 운영체제에서 실행할 수 있으며 사용할 수 있는 메모리의 양이 제한됩니다.

  - x64 모드는 64비트 환경에서 실행되는 프로그램을 만들기 위한 모드입니다. 64비트 운영체제에서 실행되며 더 많은 메모리를 사용할 수 있고 성능이 향상될 수 있습니다.

## **3. 빌드 전에 비주얼 스튜디오에서 포함 디렉토리를 설정하는 이유를 설명하라.**
  - 컴파일러가 소스 코드에서 #include문을 만날 때, 지정된 디렉토리를 순서대로 검색하여 헤더 파일을 찾을 수 있게 합니다.  
기본 경로 외에 외부 라이브러리나 프로젝트별 헤더를 사용하려면 추가 경로를 명시해야 합니다.  
설정하지 않으면 컴파일 헤더 파일을 찾을 수 없다는 오류가 발생합니다.
  
## **4. 빌드 전에 비주얼 스튜디오에서 라이브러리 디렉토리를 설정하는 이유를 설명하라.**
  - 라이브러리 디렉토리는 링커가 정적 라이브러리(.lib) 파일을 검색할 경로를 지정합니다.  
링커가 빌드 시 .lib 파일을 찾아 함수나 변수를 연결해야 하며, 기본 경로 외에 외부 라이브러리를 사용하려면 추가 경로를 명시합니다.  
설정하지 않으면 링크 오류가 발생합니다.

## **5. 빌드 전에 비주얼 스튜디오에서 링커 추가 종속성을 설정하는 이유를 설명하라.**
  - 링커 추가 종속성은 링커가 특정 .lib 파일을 명시적으로 포함하도록 지정합니다.  
라이브러리 디렉토리에서 파일을 찾더라도 링커가 자동으로 모든 .lib를 연결하지 않으므로 필요한 파일을 직접 나열해야 합니다.  
외부 라이브러리의 함수를 호출할 때 unresolved symbol 오류를 방지합니다.

## **6. DLL를 사용하기 위하여 시스템 환경변수 Path에 추가하는 경로는 무엇이고 왜 추가해야 하는지 설명하라.**
  - DLL을 사용하기 위해 환경변수 Path에 추가하는 경로는 DLL 파일이 저장된 폴더의 경로입니다.  
프로그램 실행 시 OS가 DLL을 로드할 때, PATH 환경변수를 검색하여 DLL을 찾습니다.  
기본 경로 외에 사용자 DLL이 다른 디렉토리에 있으면 OS가 해당 DLL을 찾지 못해 프로그램 실행 오류가 발생합니다.
